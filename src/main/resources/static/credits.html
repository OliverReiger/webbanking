Spring BootDas Spring Framework ist ein quelloffenes Framework für die Java-Plattform, welches oft für Web-Anwendungen verwendet wird. Ziel des Spring Frameworks ist es, die Entwicklung mit Java/Java EE zu vereinfachen und gute Programmierpraktiken zu fördern.
AnnotationenAnnotationen werden für SpringBoot Code zur Steuerung und Programmierung verwendet.
Bean-LifecycleDurch die technische Kenntnis des Bean-Lifecycle haben wir die Möglichkeit mit entsprechenden Annotationen in bestimmten Zeitpunkten bei der Erstellung und Zerstörung einer Bean mit eigenen Methoden aktiv zu werden.
@SpringBootApplicationDiese Annotation kennzeichnet, dass es sich um eine SpringBoot-Anwendung handelt.
@Component@Component ist eine Anmerkung, die es Spring ermöglicht, unsere benutzerdefinierten Beans automatisch zu erkennen.
@ServiceDiese Annotation wird für eine Klasse verwendet. Der Service markiert eine Java-Klasse, die einen Dienst ausführt, z. B. Geschäftslogik
@Controller@Controller wird verwendet, um anzugeben, dass es sich bei der Klasse um einen Spring-Controller (für eine View) handelt.
@RestControllerDiese Annotation wird auf Klassenebene verwendet. Der @RestController markiert die Klasse als Controller, bei dem jede Methode ein Domänenobjekt anstelle einer Ansicht zurückgibt.
@RepositoryDiese Annotation wird für Java-Klassen verwendet, die direkt auf die Datenbank zugreifen. Sie fungiert als Marker für die Klasse, die die Rolle eines Repositorys oder Datenzugriffsobjekts erfüllt.
@EntityBei einer Entity handelt es sich um einen Datensatz. Die passende Annotation kennzeichnet dies.
@ColumnKennzeichnet eine Variable einer Klasse als Spalte in einer Datenbanktabelle.
@IDKennzeichnet in Verbindung mit der Datenbank eine ID.
@ManyToOneKennzeichnet in Verbindung mit Datenbanken eine ManyToOne Verbindung zwischen Tabellen.
@OneToManyKennzeichnet in Verbindung mit Datenbanken eine OneToMany Verbindung zwischen Tabellen.
@QueryKennzeichnet in einer Repository-Klasse SQL Statements für die DB Abfragen.
@PathVariableDiese Annotation wird zum markieren von Methodenargumenten des Anforderungshandlers verwendet.
@RequestParamDiese Annotation wird ebenfalls zum markieren von Methodenargumenten des Anforderungshandlers verwendet.
@GetMappingDiese Annotation wird verwendet, um HTTP-GET-Anforderungen bestimmten Handler-Methoden zuzuordnen.
@DeleteMappingDiese Annotation wird verwendet, um HTTP-DELETE-Anforderungen bestimmten Handler-Methoden zuzuordnen.
@PostMappingDiese Annotation wird verwendet, um HTTP-POST-Anforderungen bestimmten Handler-Methoden zuzuordnen.
@PutMappingDiese Annotation wird verwendet, um HTTP-PUT-Anforderungen bestimmten Handler-Methoden zuzuordnen.
@RequestMappingDiese Annotation wird sowohl auf Klassen- als auch auf Methodenebene verwendet. Sie wird verwendet, um Webanforderungen bestimmten Handler-Klassen und Handler-Methoden zuzuordnen.
@ResponseBodyDiese Annotation wird verwendet, um Request-Handler-Methoden zu kommentieren. Der @ResponseBody ähnelt der @RequestBody Annotation.
@AutowiredDie @Autowired Annotation fügt implizit Objektabhängigkeit ein.
@PostConstructKennzeichnet eine Methode die im Bean-Lifecycle vor dem Abschluss des Constructors ausgeführt wird.
@PreDestroyKennzeichnet eine Methode die im Bean-Lifecycle kurz vor dem zerstören eines Objekts ausgeführt werden sollen.
@ValueDiese Annotation gibt einen Standardwertausdruck für das Feld oder den Parameter an, mit dem die Eigenschaft initialisiert werden soll.
@BeforeEachAnnotation für den Test. Diese Methode wird vor Beginn jedes Tests angewendet.
@TestAnnotation für Methoden die dem Test dienen.
@AfterEachAnnotation für den Test. Das Gegenstück zu BeforeEach. Diese Methode wird nach Ende jedes Tests angewendet.
DatenbankenEine Datenbank ermöglicht die Persistierung von Daten.
MariaDBMariaDB ist ein freies, relationales Open-Source-Datenbankmanagementsystem, das durch eine Abspaltung aus MySQL entstanden ist.
SQLSQL ist eine Datenbanksprache zur Definition von Datenstrukturen in relationalen Datenbanken sowie zum Bearbeiten (Einfügen, Verändern, Löschen) und Abfragen von darauf basierenden Datenbeständen.
SELECTBefehl zur Suche nach etwas in einer SQL-Datnbank.
INSERTBefehl zum Einfügen von Daten in einer SQL-Datnbank.
UPDATEBefehl zur Aktualisierung von Daten in einer SQL-Datnbank.
DELETEBefehl zum Löschen von Daten in einer SQL-Datnbank.
JOINSMöglichkeit Daten aus mehreren Tabellen in einer Ausgabe zu verknüpfen.
LEFT_JOINZur Tabelle A wird eine Schnittmenge aus einer Tabelle B dazugefügt.
RIGHT_JOINZu einer Schnittmenge Tabelle A wird die komplette Tabelle B dazugefügt.
INNER_JOINLediglich Schnittmengen von Tabelle A und Tabelle B werden ermittelt und ausgegeben.
ALTERBefehl um an der Struktur von Tabellen einer Datenbank etwas zu ändern.
ADD COLUMNZusätzliche Spalte hinzufügen. In Kombination mit ALTER möglich.
CREATE TABLEErzeugt eine neue Tabelle.
VIEWViews sind extra erzeugte Ansichten von einer oder mehreren Tabellen.
CREATE VIEWErzeugt eine neue View.
WHEREMit einer WHERE-Klausel können in Verbindung mit einem SELECT Kriterien zur Eingrenzung der Datenmenge gemacht werden.
GROUPBYHierbei handelt es sich um eine Möglichkeit die ermittelte Datenmenge aus dem Select zu gruppieren.
SORTMit Sort können die ermittelten Datensätze aus einem SELECT z.B. auf- oder absteigend sortiert werden.
ALIASSQL bietet die Möglichkeit Tabellennamen, Spalte, etc. innerhalb des Statements einem Alias zu geben. Dies kann gerade in Verbindung mit JOINS interessant sein.
auto_incrementAktiviert man diese Option dann vergibt die Datenbank selbst IDs für neue Einträge.
ConstraintVerknüpft Daten zwischen Tabellen.
FOREIGN KEYMit einem Foreign Key kann eine Verknüpfung erstellt werden, die beim Löschen prüft ob diese überhaupt erlaubt ist.
